Chitter - a Twitter clone
Solo project
A ruby application created as a weekend challange at the 4th weekend of Makers Academy.
Sign up - log in - write peeps - comment peeps - log out.
Front-end: CSS, HTML,
Back-end: Ruby, SQL,
Testing: Rspec,
Database: Postgresql,
Framework: Sinatra

How to run it:

ONLINE:
- tweet-chit.herokuapp.com

LOCALLY:
- Clone this repository into to your local computer.
- $ gem install bundle
- $ bundle
- $ rackup
- visit http://localhost:9292/

Design of code:

Class User

methods:

  • @id - id of a user.
  • @name - name of a name.
  • @username - username of a user.
  • @email - email of a user.
  • @password - password of a user.
  • self.create - puts a user to a database if data are valid.
  • self.email_in_use?(email) - checks if email is already in the database.
  • self.username_in_use?(username) - checks if username is already in the database.
  • self.self.find_user_by_id(id) - gets a user from a database according to a given id.
  • self.valid? - checks is length of provided sing up data is long enough.
  • self.instanciate(username) - creates a new instance of User class.
  • self.matching_data(username, password) - checks if username matches the password.

    • Class Peep

    methods:

  • @id - id of a peep.
  • @text - text of a peep.
  • @author - author of a peep.
  • @date - date of peep.
  • self.all - gets all peeps from the database.
  • self.create(text, author) - creates a new peep and saves it in the database.
  • self.in_limit?(text) - checks if a peep is not too long.

    • Class Flash

    methods:

  • self.welcome(name) - welcomes a user
  • self.long_peep - tells that peep is too long
  • self.no_name - tells that name wasn't provided
  • email_in_use - tells that email is in use
    •
  • self.username_in_use - tells that username is in use
  • self.too_short - ask to type at least 4 characters in sign up
  • self.after_log_in(name) - welcomes after log in
  • self.no_match - tells that username doesn't match password
  • self.after_log_out - says bye after log out
  • self.thanks_for_comment - thanks for commenting
  • self.no_id - tells that there is no peep with a given id
    •

    Class DatabaseConnection

    methods:

  • self.setup(database) - creates a connection using PG gem
  • self.connection - returns the connection
  • self.query(sql_string) - a skeleton for writing SQL commands

    • User Stories:


    As a Maker
    So that I can let people know what I am doing
    I want to post a message (peep) to chitter

    As a maker
    So that I can see what others are saying
    I want to see all peeps in reverse chronological order

    As a Maker
    So that I can better appreciate the context of a peep
    I want to see the time at which it was made

    As a Maker
    So that I can post messages on Chitter as me
    I want to sign up for Chitter

    As a Maker
    So that only I can post messages on Chitter as me
    I want to log in to Chitter

    As a Maker
    So that I can avoid others posting messages on Chitter as me
    I want to log out of Chitter

    As a Maker
    So that I can enjoy Chitter more
    I want to comment the Peeps
